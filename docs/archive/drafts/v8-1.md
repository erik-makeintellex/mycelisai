Evolve V8 from a structured, interactive AI system into a **persistent, event-capable, policy-bounded intelligence runtime**.

* Promotion of Response Contract and Agent Type Profiles to first-class architecture

# 🧩 Architecture Layers (Detailed)

## 1. Loop Profiles (NEW — REQUIRED)

### Definition

A Loop Profile defines **persistent execution behavior** within an AI Organization.

### Types

#### Scheduled Loop

* trigger: cron / interval
* example: daily report

#### Event Loop

* trigger: event bus (NATS)
* example: API callback, system event

#### Review Loop

* trigger: completion of another process
* example: QA reviewing outputs

#### Actuation Loop (restricted)

* trigger: event or schedule
* example: hardware/system action
* requires strict policy gating

---

### Loop Profile Schema (initial)

```yaml
id: string
name: string
type: scheduled | event | review | actuation

owner:
  type: team | agent_type
  id: string

trigger:
  schedule: optional cron
  event_subject: optional string

inputs:
  sources: [events | memory | external]

actions:
  allowed_capabilities: []

policy:
  requires_approval: boolean
  audit_required: boolean

state:
  persistence: short | long
```

---

## 2. Runtime Capabilities (NEW — REQUIRED)

### Definition

Defines what actions an entity can perform.

### Categories

* API access
* filesystem
* browser automation
* MCP/tooling
* hardware control
* messaging/event emission

---

### Capability Model

```text
Organization
  → Role (Kernel / Council)
    → Team
      → Agent Type
        → Agent Instance
```

### Rules

* allowlist only
* deny by default
* must be auditable
* must be visible in advanced UI

---

## 3. Response Contract (PROMOTED)

### Definition

Defines output behavior constraints.

### Required Properties

* tone
* verbosity
* structure
* formatting rules
* safety posture

### Inheritance

```text
Organization → Team → Agent Type → Agent Instance
```

---

## 4. Agent Type Profiles (PROMOTED)

### Definition

Reusable definition of specialist behavior.

### Required Fields

```yaml
id: string
name: string
description: string

engine:
  default: string
  override_allowed: boolean

response_contract:
  default: string
  override_allowed: boolean

capabilities:
  required: []
  optional: []

policy:
  mutable: boolean
```

# 🚀 Release Target (CRITICAL)

The system is considered **release-ready** when:

### MUST HAVE

* allow unrestricted tool use
* break bundle-driven startup
