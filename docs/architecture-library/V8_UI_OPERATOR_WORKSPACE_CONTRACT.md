# V8 UI Operator Workspace Contract
> Navigation: [V8 UI/API Contract](V8_UI_API_AND_OPERATOR_EXPERIENCE_CONTRACT.md)

Status: canonical workspace detail.

## First-Run Flow

The first-run path should teach that Mycelis creates AI Organizations, not disposable chat threads.

Required visible concepts:
- AI Organization name/purpose
- Soma
- Team Lead support
- starter template or empty-start choice
- AI Engine Settings and Memory & Continuity as guided concepts

## Organization Re-Entry

When organizations exist, default to the last active or selected organization. The primary action should open Soma workspace, not start an anonymous chat.

## Soma Workspace

Soma workspace must support:
- direct answers
- proposal state for protected actions
- execution results
- blockers and retry guidance
- retained output references
- activity/run visibility

## Teams And Groups

Team/group surfaces should be compact and reviewable:
- broad asks become smaller lanes
- Team Lead focus stays visible
- retained outputs remain accessible after closure/archive
- communication/status/result history remains inspectable

## Advanced Boundaries

Advanced mode may show:
- provider policy
- response style
- inheritance/source details
- capability posture
- templates/config origin

It must not be required for a beginner to create and use an AI Organization.
